# Task 02 — 定義受眾（persona + 逐節大綱）

## 目標
產出三份受眾規格檔，作為後續草稿的**唯一結構依據**：
`content/audiences/developer.md`、`content/audiences/power-user.md`、`content/audiences/general-public.md`。
每份 = persona 設定 + 逐節大綱（章節/頁/內容重點/圖/來源）+ 用詞政策 + 三問題對應。

## 輸入
- `docs/PLAN.md` §5（內容矩陣 + §5.1/§5.2/§5.3 三張大綱表）
- `docs/GLOSSARY.md`（§4，含每詞「用於 D/P/G」欄）
- `content/knowledge-base.md`（確認大綱引用的事實都存在）
- `sources/source-index.md`（§2，來源編號）

## 不變式（AGENTS.md）
- #6 用詞一致：詞彙政策必須引用 GLOSSARY。
- #7 語言：繁體中文，術語首次附英文。
- #8 受眾隔離：這三份規格**就是**隔離層；深度/詞彙差異只在此定義，不動 KB 事實。

## 步驟
1. 為每個受眾寫 **persona 區塊**，至少含：
   - 假設背景（讀者已知什麼）
   - 詞彙層級（依 GLOSSARY「用於」欄：哪些詞可直接用、哪些須改成類比）
   - 深度（技術細節到哪一層）
   - 篇幅目標（開發者 28–40 / 使用者 14–22 / 大眾 12–18 頁）
   - 語氣
   - 禁用清單（例：大眾版禁所有技術名詞）
2. 把 §5.x 的逐節大綱表（# / 章節 / 頁 / 內容重點 / 圖 / 來源）**完整轉錄**進對應檔，作為草稿必須遵循的權威結構。
3. 加 **三問題對應**：標明哪些章節回答「為何要做 / 如何做到 / 做到代表什麼」。
4. 加 **用詞政策**：依 GLOSSARY 列出「可直接使用的詞」與「須以類比替換的詞 + 建議類比」。
5. 交叉檢查：大綱引用的每條事實都能在 `knowledge-base.md` 找到對應 `[S0X]`；缺者回報，不自行補事實。

## 完成定義（逐項勾選）
- [ ] 三份檔皆存在，各含 persona + 逐節大綱 + 用詞政策 + 三問題對應。
- [ ] 頁數目標與章節數與 §5 一致。
- [ ] 用詞政策與 §4 GLOSSARY 的「用於」欄一致。
- [ ] 大綱所引事實皆可對應 KB；無法對應者已列出待 Phase 1 補。

## 禁止
- 不得撰寫面向讀者的正文（這是規格階段）。
- 不得引入 KB 以外的事實。
- 不得擅自更動頁數目標或章節順序而未註明理由。
