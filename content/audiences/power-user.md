> 受眾隔離層（AGENTS #8）。與開發者版**同一批事實**（`knowledge-base.md`），只是更淺的框架。
> 不新增事實；草稿 `drafts/02-power-user.md` 須遵循本結構與用詞政策。

## Persona
- **假設背景**：每天用 ChatGPT／Claude／Gemini 等 AI 助理。懂「雲端」「加密」「帳號」等大眾概念，但**非安全/密碼學專業**。核心關切：「我打進去的東西，到底誰看得到？」
- **詞彙層級**：採 GLOSSARY 的 **P 政策**——技術名詞改用白話對應（見下）。可提及 Secure Enclave 但須附一句白話；不出現 RFC 編號、暫存器名、daemon 名。
- **深度**：概念 + 類比 + 實用對照，不進協定/暫存器層。
- **篇幅目標**：**14–22 頁**。
- **語氣**：友善、清楚、實用；像一位懂技術的朋友幫你判斷。
- **禁用清單**：不整段堆術語而不解釋；類比**不得誇大**超過 KB（例：別暗示「絕對永遠不可能被看到」）。

## 用詞政策（依 GLOSSARY「用於 P」）
| KB 術語 | 本版改用 |
|---|---|
| Attestation | 認證（節點證明自己跑的是公開的那套軟體） |
| Transparency log | 人人可查的紀錄 |
| OHTTP relay | 匿名轉送 |
| Non-targetability | 沒辦法被「點名」鎖定 |
| Enforceable guarantees | 技術上做不到，而不只是「保證不做」 |
| Stateless computation | 用完即丟、不留底 |
| Verifiable transparency | 外人可以親自查驗 |
- **可直接使用**（附一句白話）：Private Cloud Compute（私有雲端運算）、Secure Enclave（晶片裡管鑰匙的保險箱）。
